
Each weekly entry should be short but meaningful, showing what you worked on, how you solved problems, and how you used resources/tools to push the project forward.

Use the following prompts to help you with your weekly entry:

<br>
<br/>

 **1. Aims**

- Goals for the week. Specific tasks, features or research topics.

Example: 
*“This week I aim to complete user authentication and set up the database schema.”*

- Add justification for why these tasks matter. Include a link to your project requirements or next steps if possible.

Example: 
*“Authentication is essential before I can personalise the dashboard.”*

<br>
<br/>

 **2. Progress**

Describe what you actually did, focusing on thinking, not just doing:

- Key tasks completed

- Code or design decisions made

- New concepts learned. (Include Evidence i.e. screenshots or snippets)

<br>
<br/>

 **3. Challenges and Solutions**

Describe problems, bugs, or blockers you faced. Explain how you diagnosed the issue (debugging, console logs, reading errors)

List resources used to help solve the problem:

- Online Resources (GitHub, StackOverflow, YouTube etc..)

- Course Notes

- ChatGPT Assistance 

- Collaboration with Peers or Teacher Assistance

Summarise the final solution and why you chose it.

<br>
<br/>

 **4. Reflection**

- What went well this week?

- What didn’t go well?

- What would you do differently next time?

- What skills or knowledge improved?

Example:
*“I underestimated how long building authentication would take. Next week I will break tasks into smaller items and estimate time more realistically.”*

<br>
<br/>

 **5. Aims for Next Week** 

Briefly end each entry by listing:

- Priorities for the next stage of development

- Parts that need more follow-up

Example:
*“Next week I will be focussing on the Quality Assurance Document and adding lazy loading to my project.”*

<br>
<br/>

# **Example Entry** - Week 4 – Backend Development

**Aims for the Week:**
My goal was to implement user authentication and have the user enter a Username/Password. I planned to use a number of resources like W3Schools and Youtube to find out how I could do this. 

**Progress:**
I watched a few videos and built an understanding for how I could create login and signup pages. I wrote the program and tested the authentication's functionality.

**Problem-Solving:**
The signup function was not functioning as expected and was not saving the Usernames correctly. I revisited the video I was using as a guide and asked a classmate to review the code. We couldn't spot the error. I then chose to ask ChatGPT to explain why the code wasn't functioning as expected. ChatGPT highlighted a small syntax error in my code that was casuing the issue. I corrected the mistake and the login feature started working smoothly. 

**Reflection:**
I made good progress this week but I spent too long troubleshooting. I need to be more careful when typing to avoid silly mistakes. One really valuable tool I learned about was the console log. This was really helpful during debugging and it will be an asset moving forward. 

**Aims for Next Week:**
Start building the user dashboard.
