As part of this project, you must document the tools you utilised, explain the rationale behind their selection (justification), and provide annotated evidence demonstrating their use. This process can be made easier by following a few key tips:

**Make sure you do this:**

1. First, document tools as they are used throughout the project rather than waiting until the end. This ensures accuracy and simplifies the task.
2. Be specific in your documentation, clearly stating the tool's name and version - the URL it is is online.
3. Using a consistent format, such as a table with columns for tool name, version, and purpose, will improve organisation and readability. For example, this table is for recording software tools used.
4. When justifying your tool selection, explain the rationale behind each choice, considering factors like specific features, ease of use, cost, compatibility, and prior experience. Connect the tool's use to specific project requirements or challenges.
5. Finally, provide annotated evidence of tool usage, primarily through screenshots. Annotate these screenshots with callouts, arrows, and text boxes to highlight key features and provide context.
For more complex tools, consider using short screen recordings with voiceover explanations. For instance, a screenshot of a flowchart in Draw.io could be annotated to show decision points, while a code screenshot in Visual Studio Code could highlight a specific function.

By following these guidelines, you can create clear, concise, and compelling documentation that effectively demonstrates your understanding of tool selection, justification, and application.
