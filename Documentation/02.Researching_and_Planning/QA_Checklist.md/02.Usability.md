# Usability – Is it user-friendly?

---

### UI/UX is clear, intuitive and easy to navigate

|What Does This Mean For The Project?|How and When Will This be Checked?|
|---|---|
|The design of the UI must be effective and efficient for a user to utilise|Will send templates of the UI to the client to check if the UI is effective for someone completely new and in flight. This will occur before the UI is designed and any changes required by the client will result in a new diagram of the UI. This will be repeated until the UI is approved by the client and is deemed achievable and realistic for me to code.|
|The design must be large enough and easy to use whilst in the air. This means that the sizing should be pretty large such that the user doesnt need to pay too much attention to the screen and visible from a distance.|This can once again be checked in flight or on the ground by the client before the designing stage and after as well according to standards set by the client.|
|The size should be adjustable according to the device used as it may be a mobile phone, tablet/iPad, or a laptop.|This can be tested through the use of various devices after coding the UI.|
|The UI should be designed such that it is easy to navigate (intuitive) such that it doesn't distract the pilot mid-flight. This will require rigorous testing and back-and-forth with the client.|This can be tested by the client and myself after the creation of the pages. Any required changes stated by the client will be done until the UI is staisfactorily navigatable.|
