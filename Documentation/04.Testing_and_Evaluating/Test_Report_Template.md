# 2. Testing Strategy

**2.1 Types of Testing**

- Unit Testing: Testing individual components or functions. 
- Integration Testing: Ensuring different modules work together. 
- System Testing: Verifying the entire system's behavior. 
- User Acceptance Testing (UAT): Ensuring the system meets user requirements. 
- Performance Testing: Checking page load speed and system response time. 
- Security Testing: Ensuring protection against vulnerabilities like SQL Injection or Cross-Site Scripting (XSS). 

<br>

**2.2 Testing Environment**

- Browsers: Chrome, Firefox, Safari, Edge
- Devices: Mobile, Tablet, Desktop
- Tools: Postman (for API testing), SQL Console (for database verification), Browser DevTools (for debugging)
